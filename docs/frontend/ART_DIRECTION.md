# ART_DIRECTION.md

# Project Axiom
## Art Direction

---

# Purpose

This document defines the visual language of Axiom.

It is not a list of UI components.

It is the visual philosophy that every design decision must follow.

Every future screen, animation, layout, and interaction should align with this document.

If a design decision contradicts this document, the document takes priority.

---

# Creative Vision

Axiom is not a developer portfolio.

It is an editorial digital experience that represents the thinking process of an engineer.

Visitors should feel they are exploring a carefully crafted engineering journal rather than browsing a personal website.

The experience should communicate craftsmanship before competence.

Thinking before technology.

Quality before quantity.

Curiosity before confidence.

---

# Emotional Journey

The experience should guide visitors through these emotions.

Curiosity

↓

Interest

↓

Respect

↓

Trust

↓

"I want this person on my team."

The interface should never interrupt this journey.

---

# Brand Personality

The product should feel:

• Curious
• Calm
• Confident
• Editorial
• Intelligent
• Crafted
• Timeless
• Precise
• Intentional

Never:

• Loud
• Flashy
• Futuristic
• Generic
• Dashboard-like
• AI generated
• Trend driven
• Over designed

---

# Visual Language

Think in materials rather than colors.

Imagine the experience is built from:

• Charcoal
• Soft graphite
• Brushed aluminum
• Warm paper
• Frosted glass
• Matte surfaces

Avoid visual noise.

Everything should feel engineered.

# Typography

Typography is the primary design element.

Typography creates hierarchy.

Typography creates confidence.

Typography creates luxury.

Avoid relying on color or decoration.

Use:

• size
• spacing
• rhythm
• weight

to create emphasis.

---

# Layout

Design pages like chapters.

Not dashboards.

Every section should feel like turning the next page of a beautifully designed book.

Whitespace is content.

Never fill space because it feels empty.

---

# Navigation

Navigation should feel like a precision-engineered object.

Not a floating widget.

Not a UI trend.

It should quietly guide the visitor.

It should almost disappear into the experience.

Navigation should feel:

• tactile
• intentional
• minimal
• balanced

---

# Components

Components should feel designed rather than assembled.

Avoid visual clutter.

Every radius.

Every spacing value.

Every border.

Every shadow.

Every animation.

must belong to the same visual system.

---

# Motion

Motion should never exist for decoration.

Every animation should answer one question:

"What is this movement helping the visitor understand?"

Motion should feel:

• cinematic
• smooth
• subtle
• natural

Avoid:

• bouncing
• spinning
• excessive parallax
• unnecessary reveals

Visitors should remember the experience.

Not the animation.

---

# Imagery

Imagery should support the narrative.

Never decorate the page with random illustrations.

Prefer:

• subtle textures
• architectural photography
• engineering details
• editorial imagery

If an image does not strengthen the story,

remove it.

---

# Color Philosophy

Think in emotion.

Not HEX values.

The interface should feel like:

Late evening.

A quiet design studio.

A premium notebook.

A Leica camera.

A Porsche interior.

Warm.

Confident.

Refined.

Never bright for attention.

Brightness should be intentional.

---

# Writing Style

Avoid marketing language.

Avoid buzzwords.

Avoid self-promotion.

Write like an engineering journal.

Confident.

Calm.

Thoughtful.

Use short sentences.

Let the work speak.

---

# Interaction Philosophy

Every interaction should have purpose.

Hover states should feel natural.

Scrolling should reveal.

Transitions should guide.

The visitor should never think:

"Nice animation."

Instead they should think:

"That felt good."

---

# Design Principles

Less, but meaningful.

Precision over decoration.

Clarity over complexity.

Story over sections.

Consistency over novelty.

Restraint over excess.

Timeless over trendy.

---

# Success Criteria

The finished experience should not feel like:

• a React portfolio
• a template
• an AI-generated website
• a collection of components

It should feel like a carefully designed product.

Visitors should leave remembering the person behind the work rather than the technologies used.

The experience should quietly communicate:

Curiosity.

Craftsmanship.

Engineering precision.

Intentional thinking.

---

# Final Principle

Before approving any design, ask:

Does this feel like Axiom?

If the answer is uncertain,

keep refining.
